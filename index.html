Data Structures and Algorithms

A developer's ability to efficiently write scalable code relies on their foundational knowledge of data structures and algorithms. I learned about many different data structures (arrays, linked lists, trees, hash tables, and graphs) and the different types of algorithms to manipulate these data structures in CS 260: Data Structures and Algorithms.

I applied these principles when adding algorithms within my capstone project in the iOS Inventory Management Application to enhance the application's functionality and performance. There are several ways I used algorithms to improve my application, such as incorporating algorithms to conduct string matching when implementing real-time search/filtering features for managing the inventory items and provide the user with the ability to sort by categories such as name, quantity, price, and added date by implementing comparison-based sorting algorithms. Additionally, I included data validation algorithms in the inventory management system to ensure that the input data was accurate and maintained integrity (duplicates excluded, data types validated, and business rules enforced). My understanding of how to evaluate an algorithmic complexity will allow me to evaluate trade-offs regarding the time and space efficiency of my applications when writing code.

Database Management

Databases are critical to supporting modern applications by providing the necessary means to store and manage data. My training in Database Administration and Advanced Programming has given me an excellent foundation for working with both relational and non-relational database systems.

The database for my inventory management application contains many connected entities ‚Äî Items, Categories, Users, Branches, Suppliers, Audit Logs, and Item Images. Data integrity is ensured via complete support for CRUD operations with appropriate transaction control. The application provides capabilities for multiple-location inventory management, for tracking item transfers between locations, and for maintaining audit logs. My implementation of a backup and restore capability reflects my understanding of best practices for managing data.

Security

Security is a fundamental requirement that is taken into account throughout the software development process. CS 405: Secure Coding has taught me how to be proactive when developing software. I anticipate that security vulnerabilities will exist and therefore, I take steps to mitigate them.

In my inventory management application, multiple levels of security are in place. The application uses password hashing to authenticate users via password without storing passwords in plain text. There are different levels of access for users based on role (Admin, Manager, Staff, Viewer). There is support for biometric authentication via Face ID and Touch ID to enhance both security and user convenience. There is input validation throughout the application to protect against injection attacks, and all sensitive data is encrypted when stored. Finally, an extensive auditing capability ensures that all significant actions will be properly tracked and will provide the basis for supporting accountability and forensic analysis.
