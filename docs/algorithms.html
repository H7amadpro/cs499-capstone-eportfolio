Artifact Description

The artifact for the enhancement is the iOS Inventory Management Application, which was previously enhanced for software design and engineering in CS 360: Mobile Architecture and Programming. The original version of the application had a limited amount of algorithmic complexity, using only basic linear searches to locate items and having no sorting capabilities beyond the default order of items in the database. Additionally, there were no sophisticated data validation algorithms.

The original implementation had limited performance because it simply iterated through the arrays in order to locate items. Therefore, this implementation of the application would have worked well for small data sets, but as the inventory increased in size, the performance of this implementation would have significantly decreased. The lack of regard for algorithmic complexity or optimization also contributed to this low performance.

Justification for Inclusion

I chose this artifact for my Algorithms and Data Structures enhancement since it provided an excellent opportunity to show my ability to apply algorithmic thinking to efficiently solve real-world problems. Since inventory management systems manage potentially large datasets while displaying responsive UIs, algorithmic efficiency is a critical factor.

The various points in the application that demonstrate my skill in the application of algorithmic thinking include:

Real-Time Search Algorithm

The search algorithm implemented in the inventory application is an efficient algorithm that allows users to filter the inventory items as they enter search terms. The algorithm performs case-insensitive string comparisons on the name, SKU, barcode, description, and tags of the inventory across multiple fields at the same time with debouncing implemented to minimize the number of filtering operations performed.

Multi-Criteria Sorting

The inventory application has a flexible sorting architecture that allows users to sort the inventory by multiple criteria including name, quantity, price, date added, and categories. The implementation of the sorting algorithm is via the Swift sorted(by:) method using custom comparators, achieving an O(n log n) time complexity.

Data Validation Algorithms

I created a comprehensive set of validation algorithms in the ValidationService class that ensure the integrity of the inventory data prior to being stored in a permanent storage location. These validation algorithms check for required fields in the inventory data, string length constraints, numeric range constraints, format constraints, and duplicate values by using the hashtable for lookups.

Binary Search of Sorted Items

The AlgorithmUtilities class implements the binary search for looking up items in sorted data sets. The binary search has an O(log n) lookup time vs. the O(n) linear search, making it a much more efficient algorithm to use to look up items in large sorted lists.

Optimized Data Structures

The data model of the application has been designed using the most appropriate data structure. For ordered collections of items, arrays are used; for collections of tags requiring uniqueness (O(1) lookups); and for the mapping of items to categories, dictionaries are used to enable O(1) lookups.

Reflection on the Enhancement Process

The enhancement of the algorithmic portions of the application has given me a deeper understanding of how the concepts of theoretical computer science can be brought to the real world through performance improvements. Through the enhancement process, I was required to analyze the time and space complexity of the various approaches to the problem of enhancing the application, and through this, make the necessary decisions regarding trade-offs.

Challenges Encountered

One of the primary challenges I faced was balancing algorithmic efficiency with code readability. While I could have spent more time creating more complex algorithms with minimal performance gains, I was ultimately more focused on creating code that would be maintainable. Additionally, I faced a challenge in implementing efficient algorithms with the reactive nature of SwiftUI. In order to do so, I needed to carefully manage the use of computed properties and @Published properties.

Incorporation of Feedback

After receiving feedback during the code review process, I implemented memoization for some of the calculations that were called multiple times, and I implemented lazy loading for the large sets of inventory items such that only a portion of the inventory items would be retrieved from memory at any single point in time.
