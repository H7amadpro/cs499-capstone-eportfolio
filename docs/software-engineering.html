Artifact Description

The artifact I selected for this enhancement is an iOS Inventory Management Application originally developed in CS 360: Mobile Architecture and Programming during my Computer Science program at Southern New Hampshire University. The original application was created as a basic inventory tracking tool that allowed users to add, view, and delete inventory items with minimal functionality. It was built using Swift and UIKit with a simple table view interface and basic SQLite database storage.

The original artifact represented a functional but limited approach to mobile application development. It lacked proper architectural patterns, had tightly coupled code, minimal error handling, and a dated user interface. While it fulfilled the basic requirements of the original course assignment, it did not reflect the professional standards expected in industry-level software development.

Justification for Inclusion

I selected this artifact for my ePortfolio because it presented an excellent opportunity to demonstrate significant growth in software design and engineering principles. The transformation from a basic inventory tracker to a comprehensive enterprise-level application showcases my ability to architect complex software systems following industry best practices.

The specific components that showcase my skills and abilities in software development include:

MVVM Architecture Implementation

I completely restructured the application using the Model-View-ViewModel (MVVM) architectural pattern. This separation of concerns ensures that the user interface (Views) remains decoupled from business logic (ViewModels) and data models (Models). This architecture makes the codebase more maintainable, testable, and scalable.

Modern SwiftUI Framework

I migrated the entire application from the legacy UIKit framework to SwiftUI, Apple's modern declarative UI framework. This migration demonstrates my ability to adapt to evolving technologies and leverage new tools to create more efficient and maintainable code.

Modular Code Organization

The project follows a well-organized folder structure with clear separation between Models, Views, ViewModels, Services, and Utilities. Each component has a single responsibility, adhering to the SOLID principles of object-oriented design.

Comprehensive Error Handling

I implemented robust error handling throughout the application using Swift's Result type and custom error enums. The application gracefully handles unexpected situations, providing meaningful feedback to users without crashing.

Professional Documentation

Every file includes comprehensive header documentation, and complex functions include inline comments explaining the logic. This documentation practice demonstrates my understanding of collaborative development environments.

Reusable Components

I created modular, reusable UI components such as ItemRowView, BranchPickerView, and ImagePicker that can be used throughout the application, reducing code duplication and ensuring consistency.

Reflection on the Enhancement Process

The process of enhancing this artifact taught me valuable lessons about software architecture and the importance of planning before coding. When I began the enhancement, I initially attempted to modify the existing codebase incrementally. However, I quickly realized that the tightly coupled nature of the original code made this approach impractical. This led me to make the strategic decision to rebuild the application from the ground up using proper architectural patterns.

Challenges Faced

One significant challenge was implementing the MVVM pattern correctly in SwiftUI. Unlike UIKit, where MVVM is straightforward, SwiftUI's reactive nature required me to carefully consider how data flows between components. I had to learn about property wrappers like @StateObject, @ObservedObject, and @EnvironmentObject to properly manage state and ensure the UI updates reactively when data changes.

Another challenge was migrating from SQLite to SwiftData while maintaining data integrity. I had to design a comprehensive data migration strategy and thoroughly test the new persistence layer to ensure no data loss during the transition.

Feedback Incorporation

Based on feedback from my code review, I made several improvements to the initial enhancement plan. I added more comprehensive input validation in the ViewModels to prevent invalid data from reaching the data layer. I also improved the error messages to be more user-friendly and actionable.

Course Outcomes

This enhancement addresses the following course outcomes:

  • Course Outcome 1: Employ strategies for building collaborative environments that enable diverse audiences to support organizational decision making in the field of computer science. The well-documented, modularly organized codebase enables other developers to easily understand, maintain, and extend the application.
  • Course Outcome 4: Demonstrate an ability to use well-founded and innovative techniques, skills, and tools in computing practices. By adopting SwiftUI and SwiftData, I demonstrated my ability to leverage innovative tools to create modern, efficient applications.
  • Course Outcome 5: Develop a security mindset that anticipates adversarial exploits in software architecture. The software design incorporates security considerations from the ground up, with separation of concerns preventing direct access to sensitive data from the UI layer.

Conclusion

The Software Design and Engineering enhancement transformed a basic student project into a professionally architected application that demonstrates my readiness for industry-level software development. The skills showcased ‚Äî architectural design, modern framework adoption, modular development, and professional documentation ‚Äî are directly applicable to real-world software engineering positions.
